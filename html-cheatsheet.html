1. Introduction

What are markup languages?

Markup languages are used to annotate text so that computers can process and display it. HTML (HyperText Markup Language) is the standard markup language for creating web pages.

What is HTTP?

HTTP (HyperText Transfer Protocol) is the protocol used for transmitting web pages over the internet. HTTPS is the secure version.

DNS (Domain Name System)

DNS translates domain names (like example.com) into IP addresses that computers use to identify each other.

Domain names

A domain name is a unique address that identifies a website on the internet (e.g., google.com, github.com).

Browsers

Web browsers (Chrome, Firefox, Safari, Edge) are applications that retrieve and display web pages.

Hosting

Web hosting is a service that stores your website files and makes them accessible on the internet.

What is SEO?

SEO (Search Engine Optimization) is the practice of optimizing websites to rank higher in search engine results.

How the web works

When you type a URL, your browser sends an HTTP request to a server, which responds with HTML, CSS, and JavaScript files that your browser renders as a web page.

2. Your First HTML File

Tags and Attributes

HTML uses tags enclosed in angle brackets. Most tags have opening and closing pairs.

<tagname attribute="value">Content</tagname>

Case Insensitivity

HTML tags are case-insensitive, but lowercase is the standard convention.

<DIV> and <div> are the same, but use <div>

HTML Entities

Special characters that need to be encoded:

Entity Character Description
&lt; < Less than
&gt; > Greater than
&amp; & Ampersand
&quot; " Quotation mark
&nbsp;   Non-breaking space
&copy; © Copyright

HTML Comments

<!-- This is a comment and won't be displayed -->

Whitespaces

Multiple spaces and line breaks are collapsed into a single space in HTML.

<p>This    has     many     spaces</p>
<!-- Displays as: This has many spaces -->

Basic HTML Structure

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Page Title</title>
</head>
<body>
    <!-- Your content here -->
</body>
</html>

3. Basic Tags

Document Structure

<!DOCTYPE html>

Declares the document type and HTML version.

<html>

Root element of an HTML page.

<html lang="en"></html>

<head>

Contains metadata about the document.

<meta>

Defines metadata like character set, viewport, description.

<meta charset="UTF-8">
<meta name="description" content="Page description">
<meta name="keywords" content="HTML, CSS, JavaScript">
<meta name="author" content="Your Name">

<title>

Sets the page title shown in browser tab.

<title>My Website</title>

<body>

Contains the visible page content.

Text Content Tags

<h1> to <h6>

Heading tags, from most important (h1) to least important (h6).

<h1>Main Heading</h1>
<h2>Subheading</h2>
<h3>Sub-subheading</h3>

<p>

Paragraph tag.

<p>This is a paragraph of text.</p>

<br>

Line break (self-closing tag).

<p>Line one<br>Line two</p>

<hr>

Horizontal rule (divider line).

<hr>

<b> / <strong>

Bold text. Use <strong> for semantic importance.

<b>Bold text</b>
<strong>Important text</strong>

<i> / <em>

Italic text. Use <em> for emphasis.

<i>Italic text</i>
<em>Emphasized text</em>

<mark>

Highlighted text.

<mark>Highlighted text</mark>

<sub>

Subscript text.

H<sub>2</sub>O

<sup>

Superscript text.

x<sup>2</sup>

<pre>

Preformatted text (preserves spaces and line breaks).

<pre>
    This    preserves
        all   formatting
</pre>

4. Textual Tags

Grouping Text

<div>

Block-level container for grouping content.

<div class="container">
    <p>Content here</p>
</div>

<span>

Inline container for styling or grouping text.

<p>This is <span style="color: red;">red text</span>.</p>

Standard Attributes

id

Unique identifier for an element.

<div id="header"></div>

class

One or more class names for styling.

<p class="important highlight">Text</p>

style

Inline CSS styling.

<p style="color: blue; font-size: 16px;">Styled text</p>

Data Attributes

Custom data attributes for storing extra information.

<div data-user-id="12345" data-role="admin">User Info</div>

Links

<a>

Anchor tag for hyperlinks.

<!-- External link -->
<a href="https://example.com">Visit Example</a>

<!-- Link in new tab -->
<a href="https://example.com" target="_blank">Open in new tab</a>

<!-- Email link -->
<a href="mailto:email@example.com">Send Email</a>

<!-- Phone link -->
<a href="tel:+1234567890">Call Us</a>

<!-- Anchor link (same page) -->
<a href="#section1">Go to Section 1</a>

5. Lists and Types

Ordered Lists

Numbered list using <ol>.

<ol>
    <li>First item</li>
    <li>Second item</li>
    <li>Third item</li>
</ol>

Unordered Lists

Bulleted list using <ul>.

<ul>
    <li>Item one</li>
    <li>Item two</li>
    <li>Item three</li>
</ul>

Definition Lists

List of terms and descriptions.

<dl>
    <dt>HTML</dt>
    <dd>HyperText Markup Language</dd>

    <dt>CSS</dt>
    <dd>Cascading Style Sheets</dd>
</dl>

Nested Lists

<ul>
    <li>Item 1
        <ul>
            <li>Sub-item 1.1</li>
            <li>Sub-item 1.2</li>
        </ul>
    </li>
    <li>Item 2</li>
</ul>

6. Table Tag

Basic Table Structure

<table>
    <thead>
        <tr>
            <th>Header 1</th>
            <th>Header 2</th>
            <th>Header 3</th>
        </tr>
    </thead>
    <tbody>
        <tr>
            <td>Row 1, Cell 1</td>
            <td>Row 1, Cell 2</td>
            <td>Row 1, Cell 3</td>
        </tr>
        <tr>
            <td>Row 2, Cell 1</td>
            <td>Row 2, Cell 2</td>
            <td>Row 2, Cell 3</td>
        </tr>
    </tbody>
</table>

Table Tags

  • <table> - Defines a table
  • <thead> - Table header section
  • <tbody> - Table body section
  • <tfoot> - Table footer section
  • <tr> - Table row
  • <th> - Table header cell
  • <td> - Table data cell
  • <caption> - Table caption

Spanning Cells

<!-- Colspan -->
<td colspan="2">Spans 2 columns</td>

<!-- Rowspan -->
<td rowspan="3">Spans 3 rows</td>

7. Embedding Media

Images

<img>

Embeds an image.

<img src="image.jpg" alt="Description of image" width="300" height="200">

<figure> and <figcaption>

Semantic way to group images with captions.

<figure>
    <img src="photo.jpg" alt="A beautiful sunset">
    <figcaption>A beautiful sunset over the ocean</figcaption>
</figure>
img vs figure: Use <img> for simple images. Use <figure> when you need to associate a caption with the image or when the image is a self-contained unit of content.

Priority Hints

Control resource loading priority.

<img src="hero.jpg" alt="Hero" fetchpriority="high">
<img src="thumbnail.jpg" alt="Thumbnail" loading="lazy">

Audio

<audio controls>
    <source src="audio.mp3" type="audio/mpeg">
    <source src="audio.ogg" type="audio/ogg">
    Your browser does not support the audio element.
</audio>

Video

<video width="640" height="360" controls>
    <source src="video.mp4" type="video/mp4">
    <source src="video.webm" type="video/webm">
    Your browser does not support the video tag.
</video>

iframe

Embeds another webpage or content.

<iframe src="https://example.com" width="600" height="400"></iframe>

CSP (Content Security Policy)

Security feature to prevent XSS attacks.

<meta http-equiv="Content-Security-Policy"
      content="default-src 'self'; script-src 'self' https://trusted.com">

8. Using Forms

Basic Form Structure

<form action="/submit" method="POST">
    <!-- Form elements go here -->
</form>

Labels and Inputs

Text Input

<label for="username">Username:</label>
<input type="text" id="username" name="username" placeholder="Enter username">

Common Input Types

<!-- Email -->
<input type="email" name="email" required>

<!-- Password -->
<input type="password" name="password">

<!-- Number -->
<input type="number" name="age" min="0" max="120">

<!-- Date -->
<input type="date" name="birthdate">

<!-- Checkbox -->
<input type="checkbox" name="subscribe" id="subscribe">
<label for="subscribe">Subscribe to newsletter</label>

<!-- Radio buttons -->
<input type="radio" name="gender" value="male" id="male">
<label for="male">Male</label>
<input type="radio" name="gender" value="female" id="female">
<label for="female">Female</label>

<!-- Textarea -->
<textarea name="message" rows="4" cols="50"></textarea>

<!-- Select dropdown -->
<select name="country">
    <option value="us">United States</option>
    <option value="uk">United Kingdom</option>
    <option value="ca">Canada</option>
</select>

<!-- Submit button -->
<button type="submit">Submit</button>
<input type="submit" value="Submit">

File Uploads

<form action="/upload" method="POST" enctype="multipart/form-data">
    <label for="file">Choose file:</label>
    <input type="file" id="file" name="file" accept=".jpg,.png,.pdf">

    <!-- Multiple files -->
    <input type="file" name="files" multiple>

    <button type="submit">Upload</button>
</form>

Form Validation

<!-- Required field -->
<input type="text" name="username" required>

<!-- Pattern validation -->
<input type="text" pattern="[A-Za-z]{3,}" title="At least 3 letters">

<!-- Min/Max length -->
<input type="text" minlength="3" maxlength="20">

<!-- Email validation -->
<input type="email" required>

<!-- Number range -->
<input type="number" min="1" max="100">

Limitations

HTML Validation Limitations:

  • Client-side validation can be bypassed
  • Always validate on the server side
  • Limited customization of error messages
  • Not all browsers support all HTML5 validation features

9. Semantic Markup

Layout Tags

Semantic HTML5 elements that give meaning to your page structure.

<header>

Introductory content or navigation.

<header>
    <h1>Site Title</h1>
    <nav>...</nav>
</header>

<nav>

Navigation links.

<nav>
    <ul>
        <li><a href="/">Home</a></li>
        <li><a href="/about">About</a></li>
        <li><a href="/contact">Contact</a></li>
    </ul>
</nav>

<main>

Main content of the document (only one per page).

<main>
    <article>...</article>
</main>

<article>

Self-contained content that could be distributed independently.

<article>
    <h2>Article Title</h2>
    <p>Article content...</p>
</article>

<section>

Thematic grouping of content.

<section>
    <h2>Section Title</h2>
    <p>Section content...</p>
</section>

<aside>

Tangentially related content (sidebar, callouts).

<aside>
    <h3>Related Links</h3>
    <ul>...</ul>
</aside>

<footer>

Footer for a section or page.

<footer>
    <p>&copy; 2026 My Website</p>
</footer>

Highlighting Changes

<del>

Deleted text (strikethrough).

<p>Price: <del>$50</del> $30</p>

<s>

Text that is no longer accurate or relevant.

<s>Out of stock</s>

<ins>

Inserted text (underlined).

<p>My favorite color is <del>blue</del> <ins>red</ins>.</p>

Quotation / Citation

<blockquote>

Block-level quotation.

<blockquote cite="https://example.com">
    <p>This is a long quotation that spans multiple lines.</p>
</blockquote>

<q>

Inline quotation.

<p>He said, <q>Hello world</q> and smiled.</p>

<cite>

Citation or reference to a creative work.

<p><cite>The Great Gatsby</cite> by F. Scott Fitzgerald</p>

<abbr>

Abbreviation or acronym.

<abbr title="HyperText Markup Language">HTML</abbr>

<dfn>

Definition term.

<p><dfn>HTML</dfn> is a markup language for web pages.</p>

<address>

Contact information.

<address>
    Written by <a href="mailto:webmaster@example.com">Jon Doe</a>.<br>
    Visit us at: example.com
</address>

10. Styling Basics

Inline CSS

Styles applied directly to an element using the style attribute.

<p style="color: red; font-size: 18px; font-weight: bold;">
    Styled text
</p>
Note: Inline styles have the highest specificity but are hard to maintain. Use sparingly.

Internal CSS

Styles defined in a <style> tag within the <head>.

<head>
    <style>
        body {
            font-family: Arial, sans-serif;
            margin: 0;
            padding: 0;
        }

        h1 {
            color: #333;
        }

        .highlight {
            background-color: yellow;
        }
    </style>
</head>

External CSS

Styles defined in a separate CSS file.

<head>
    <link rel="stylesheet" href="styles.css">
</head>
Best Practice: Use external CSS files for better organization and reusability.

11. Including JavaScript

Inline JavaScript

<button onclick="alert('Hello!')">Click Me</button>

Internal JavaScript

<script>
    console.log('Hello World');

    function greet() {
        alert('Welcome!');
    }
</script>

External JavaScript

<!-- In head (blocks rendering) -->
<head>
    <script src="script.js"></script>
</head>

<!-- Before closing body (recommended) -->
<body>
    <!-- Content -->
    <script src="script.js"></script>
</body>

<!-- With defer (loads asynchronously, executes after HTML parsing) -->
<script src="script.js" defer></script>

<!-- With async (loads and executes asynchronously) -->
<script src="script.js" async></script>

12. Accessibility

Core Principles

Web accessibility ensures that websites are usable by everyone, including people with disabilities.

Best Practices

Use Semantic HTML

<!-- Good -->
<button>Click Me</button>

<!-- Bad -->
<div onclick="doSomething()">Click Me</div>

Alt Text for Images

<img src="logo.png" alt="Company Logo">
<img src="decorative.png" alt=""> <!-- Empty alt for decorative images -->

Form Labels

<label for="email">Email Address:</label>
<input type="email" id="email" name="email">

ARIA Attributes

<button aria-label="Close dialog">X</button>
<nav aria-label="Main navigation">...</nav>
<div role="alert" aria-live="polite">Error message</div>

Keyboard Navigation

<!-- Ensure interactive elements are keyboard accessible -->
<button tabindex="0">Accessible Button</button>

Color Contrast

Ensure sufficient contrast between text and background colors (WCAG recommends 4.5:1 for normal text).

Skip Links

<body>
    <a href="#main-content" class="skip-link">Skip to main content</a>
    <header>...</header>
    <main id="main-content">...</main>
</body>

13. Basics of SEO

Meta Tags

<head>
    <!-- Title (most important) -->
    <title>Page Title | Site Name</title>

    <!-- Meta description -->
    <meta name="description" content="Clear, concise page description (150-160 chars)">

    <!-- Keywords (less important nowadays) -->
    <meta name="keywords" content="keyword1, keyword2, keyword3">

    <!-- Canonical URL -->
    <link rel="canonical" href="https://example.com/page">

    <!-- Open Graph (for social media) -->
    <meta property="og:title" content="Page Title">
    <meta property="og:description" content="Page description">
    <meta property="og:image" content="https://example.com/image.jpg">
    <meta property="og:url" content="https://example.com/page">

    <!-- Twitter Card -->
    <meta name="twitter:card" content="summary_large_image">
    <meta name="twitter:title" content="Page Title">
    <meta name="twitter:description" content="Page description">
</head>

SEO Best Practices

Semantic HTML Structure

Use proper heading hierarchy (h1, h2, h3...) and semantic tags.

Clean URLs

<!-- Good -->
https://example.com/products/red-shoes

<!-- Bad -->
https://example.com/page?id=123&cat=45

Image Optimization

<img src="image.jpg"
     alt="Descriptive alt text with keywords"
     width="800"
     height="600"
     loading="lazy">

Internal Linking

Link to other pages on your site with descriptive anchor text.

Mobile-Friendly

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Page Speed

  • Minimize HTTP requests
  • Compress images
  • Use lazy loading
  • Minify CSS and JavaScript

Structured Data (Schema.org)

<script type="application/ld+json">
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Article Title",
  "author": {
    "@type": "Person",
    "name": "John Doe"
  },
  "datePublished": "2026-01-13"
}
</script>

Robots.txt

<!-- Control search engine crawling -->
<meta name="robots" content="index, follow">
